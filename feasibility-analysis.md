# 风水数字化软件 — 可行性分析报告

> 分析日期：2026-05-12  
> 目标平台：iPhone（优先），兼顾 Android / Web

---

## 一、项目概述

将中国传统风水理论（形势派 + 理气派）转化为现代化数字产品，核心命题是：

> 用手机传感器、图像识别和算法，让普通人也能做基本的风水诊断。

---

## 二、模块技术可行性详细分析

### 模块 1：AR 实景勘测

#### 1.1 虚拟罗盘（★★★★★ 完全可行）

| 维度 | 评估 |
|------|------|
| **技术方案** | Web `DeviceOrientation` API / iOS `CoreLocation` + `CMAttitude` |
| **实现难度** | 低 |
| **可行性** | PWA 即可实现，iOS 13+ 需用户手动授权 |
| **精度** | 手机电子罗盘精度 ±5°，民宅风水够用；金属干扰需做"靠近金属提醒" |
| **关键代码量** | ~200 行 JS |

**技术细节：**

```
window.addEventListener('deviceorientation', (e) => {
  // e.alpha: 0-360, 手机指向的罗盘方位
  // 转换为二十四山向: 每山 15°
  const shanIndex = Math.round(e.alpha / 15) % 24;
});
```

**结论：PWA 直接可用。**

---

#### 1.2 煞气识别 — 摄像头实时检测（★★☆☆☆ 基本不可行）

| 维度 | 评估 |
|------|------|
| **技术方案** | 目标检测模型（YOLO/RT-DETR）+ ARKit 锚点 |
| **实现难度** | 极高 |
| **可行性** | Web 几乎不可行；原生 App 勉强可行 |
| **核心问题** | 1) "尖角煞""路冲""天斩煞"需要语义理解，非简单物体检测<br>2) 室外场景复杂，误报率极高<br>3) 训练数据不存在——没有标注好"煞气"的数据集 |

**结论：当前技术不成熟，建议降级为"拍照 + 人工标注"，用户拍照后手动圈出疑似煞气，软件给出分析。**

---

#### 1.3 AR 家具摆放模拟（★★★☆☆ 部分可行）

| 维度 | 评估 |
|------|------|
| **技术方案** | Web: 2D Canvas 户型编辑器 / 原生: ARKit SceneKit |
| **实现难度** | 中 |
| **可行性** | Web 版做 2D 平面拖拽可行；AR 版需原生开发 |
| **PWA 方案** | 在户型图上用 Canvas 拖拽虚拟家具图标，软件实时判断方位并弹出建议 |

**结论：2D 版完全可行，AR 版需原生。**

---

### 模块 2：AI 户型诊断

#### 2.1 九宫飞星布局（★★★★★ 完全可行）

| 维度 | 评估 |
|------|------|
| **技术方案** | 纯算法：用户标注朝向 → 计算九宫格 → 叠加年/月飞星盘 |
| **实现难度** | 低 |
| **可行性** | 100%，不需要 AI |
| **关键代码量** | ~300 行 JS |

**算法逻辑链：**

```
坐向(度数) → 二十四山向 → 宅卦(八宅)
         ↓
    户型九宫格划分
         ↓
    年飞星盘 + 宅命盘 → 每个宫位吉凶
         ↓
    财位/文昌位/桃花位/凶位 → 布局建议
```

**九宫飞星是数学规则，不是机器学习。全部可算。**

---

#### 2.2 户型图智能识别（★★☆☆☆ 半自动可行）

| 维度 | 评估 |
|------|------|
| **技术方案** | 全自动：CV 语义分割模型；半自动：用户手动标角点 |
| **实现难度** | 全自动极高，半自动低 |
| **可行性** | 半自动标注模式下完全可行 |
| **半自动方案** | 用户上传户型图 → Canvas 显示 → 用户拖动四个角点对齐墙壁 → 系统计算面积和九宫格 → 输出诊断 |

**结论：先做半自动标注，全自动识别后续迭代。**

---

#### 2.3 缺角检测（★★★★★ 完全可行）

九宫格划分后，逐格检查面积占比。某格面积 < 标准值的 60% 即视为缺角。纯数学计算。

---

### 模块 3：环境气象大数据

| 维度 | 评估 |
|------|------|
| **技术方案** | 接入高德地图 API / Google Maps API + GIS 数据 |
| **实现难度** | 中 |
| **可行性** | 需要后端服务器 + API Key |
| **成本** | 高德地图 API 有免费额度（5000次/天），初期够用 |
| **核心功能** | 1) 周边水系/山体/道路分析<br>2) 光照模拟（SunCalc 开源库）<br>3) 通风模拟（需 CFD 建模，复杂） |

**结论：光照分析简单可行（纯前端的 SunCalc.js），水系/山体分析需地图 API。通风模拟过于复杂，不建议前期做。**

---

### 模块 4：数字化择吉避凶

#### 4.1 个性化择日历（★★★★★ 完全可行）

| 维度 | 评估 |
|------|------|
| **技术方案** | 八字排盘算法 + 万年历数据库 + 神煞规则引擎 |
| **实现难度** | 中（算法复杂但全部可算） |
| **可行性** | 100%，与周易 App 技术栈完全一致 |
| **关键数据** | 万年历宜忌表、二十四节气、干支纪日、神煞规则 |

**排盘 → 择日逻辑：**

```
用户出生年月日时
    ↓
八字四柱排盘
    ↓
找出日干 → 计算贵人日(天乙贵人/文昌贵人等)
    ↓
避开冲煞日、三煞日
    ↓
结合万年历宜忌 → 输出推荐日期
```

### 模块 5：色彩与绿植建议（★★★★★ 完全可行）

纯规则引擎：

```
方位(度数) → 所属五行 → 相生色/相克色 → 推荐色板
                              ↓
                        推荐植物（基于植物五行属性表）
```

| 方位 | 五行 | 推荐色 | 推荐植物 |
|------|------|--------|----------|
| 东 | 木 | 绿色系 | 发财树、绿萝 |
| 南 | 火 | 红色系 | 红掌、鸿运当头 |
| 西 | 金 | 白色/金色 | 白掌、银皇后 |
| 北 | 水 | 蓝色/黑色 | 水培富贵竹、龟背竹 |
| 中 | 土 | 黄色/棕色 | 金钱树、虎皮兰 |

## 四、推荐 MVP 方案

### MVP 1.0：风水诊断工具箱（PWA）

**包含功能（全部 P0 + P1）：**

| 功能 | 输入 | 输出 | 实现方式 |
|------|------|------|----------|
| **虚拟罗盘** | 手机陀螺仪 | 二十四山向读数 | DeviceOrientation API |
| **户型九宫诊断** | 上传户型图 + 手动标角 | 九宫格叠加 + 飞星吉凶 | Canvas + 飞星算法 |
| **缺角检测** | 同上 | 缺角位置 + 化解建议 | 面积计算 |
| **择日历** | 出生日期 | 个人贵人日/吉日推荐 | 八字排盘 + 万年历 |
| **五行配色** | 方位信息 | 色板 + 植物推荐 | 规则引擎 |
| **光照模拟** | 户型朝向 | 各时段光照分析 | SunCalc.js |

**技术方案：**
- 单文件 PWA（与周易 App 同一架构）
- 纯前端，零服务器
- iOS Safari "添加主屏" = 原生体验
- 总计代码量：约 2000-3000 行

**开发周期估算：** 2-3 周（全职）

## 五、技术风险与缓解

| 风险 | 严重度 | 缓解方案 |
|------|--------|----------|
| 手机罗盘精度不足 | 中 | 显示误差范围，加"远离金属"提示；允许手动微调 |
| iOS 陀螺仪权限问题 | 低 | iOS 13+ 需用户点击授权，添加引导 UI |
| 户型图识别不准 | 中 | 先做半自动标注，不做全自动 |
| 风水流派众多（八宅/玄空/三合） | 中 | MVP 先做最通用的八宅+玄空飞星 |
| 择日算法分歧 | 低 | 注明"传统算法参考"，不做绝对判断 |

## 八、结论

**核心结论：作为 PWA，MVP 1.0（罗盘 + 九宫飞星 + 择日历 + 配色建议）技术上 100% 可行，无需原生开发，可与现有周易 App 整合为一个「传统文化工具箱」。**

**建议：**
1. 先不做 AR 煞气识别（技术债太大）
2. 先做半自动户型标注（不做全自动 AI）
3. 九宫飞星 + 择日历 是算法密集型，适合纯前端实现
4. 可复用周易 App 的 PWA 架构，统一部署
